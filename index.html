The skillset can improve any target by understanding its purpose. It was aimed at its own skillset 200+ times with different AI models - improving its own ability to improve any target by understanding its own purpose to improve any target - until no model could improve anything further.

Principles of Earned Autonomy

Principle #1: Commander’s Intent ↗

“Define the destination. Never prescribe the route.”

Instructions limit autonomy. Make the AI interpret your intent and decide how to proceed. An autonomous agent must understand what needs to be achieved and why it matters.

Principle #2: Observable Autonomy ↗

“The degree of autonomy a system deserves is bounded by the degree of transparency it provides.”

Autonomous actions that cannot be audited should be revoked.

Principle #3: Convergence Is Silence ↗

“Not done until multiple independent models find nothing to change.”

Definition of when the improve loop has achieved the desired outcome.

Autonomous Reasoning Fidelity (ARF) ↗

Appears only when all three principles hold.

ARF is the signal that measures if the agent is reasoning visibly enough to earn autonomy (Principles of Earned Autonomy). Probe is the skill that tests for it.

The Skills

Install

Download the six skill files from the skills repository ↗ and place them in your agent's skills folder (.copilot/skills/ for GitHub Copilot, or wherever your toolset reads custom instructions from).

Intent SKILL ↗

Force the Agent to understand the intent behind your prompt in relation to the context: what you mean, not what you said. Cites how it reads your intent before it continues.

The principle behind military command, Socratic method, and coaching kata.

Invoke: Every prompt.

To trigger: “understand my intent”

P1 · Commander’s Intent

Trail SKILL ↗

Writes every session to .trail/audit-trail.md - the append-only source of truth. One entry per run: the ask, examination, decisions, actions, reflection. The next prompt knows what was already done and why.

Regenerates the derived views at each commit: history.md (the run timeline) and learning.md (every realization and reversal). Full summaries land in sessions/; raw dialogue in transcripts/ when the harness can export it.

Invoke: Every session.

To trigger: “trail everything”

P2 · Observable Autonomy

Improve SKILL ↗

Iterative improvement loop. Improves any target by understanding the context, purpose and intent. Applied to itself 200+ times.

One highest leverage change per run. Reads the context first - destination, retrospect, trail - acts, and logs it.

Invoke: Until .trail/destination.md is fulfilled.

To trigger: “use the improve skill”

P3 · Convergence Is Silence

Destination SKILL ↗

The agent reads the conversation and the trail and states what it thinks you're building and why, as specific questions for you to confirm or correct. Produces .trail/destination.md - where you're going: what the target is for, the constraints that hold across runs, the hard problem the work is chipping at. Operator-held; humans only.

Invoke: At the start - when direction feels unclear.

To trigger: “use the destination skill”

Retrospect SKILL ↗

Reads the entire trail. Self-reflects on all iterations and surfaces patterns no single session can see. Produces .trail/retrospect.md - where you are: arc-level orientation plus the active rules the agent must obey this run.

Invoke: After a few improvement skills iterations.

To trigger: “use the retro skill”

Probe SKILL ↗

Tests whether the agent’s response diverges where it should. Measures Autonomous Reasoning Fidelity.

Invoke: When you need to verify reasoning quality.

To trigger: “use the probe skill”

Memory Model

The skills give every repository a memory layer in .trail/, written to the root of the repo being worked on. Each run reads it before acting and extends it after - that is what makes the loop cumulative instead of repeating yesterday's mistakes.

A hierarchy resolves disagreements: the operator-held destination.md overrides the agent's orientation in retrospect.md; and where orientation and the append-only audit-trail.md evidence disagree, the trail wins. Memory without a hierarchy is just noise.

Built by the loop it describes

Without data, you're just another person with an opinion. attributed to W. Edwards Deming

These skills are the output of running the Improve loop on itself - more than 200 iterations, across the loop's own evolution. It began as Kaizen, borrowed from Toyota Kata. It split into Kaizen, Kaikaku, and Hansei. It was unified into the single Improve skill in use today. Each version was sharpened by the version before it.

Every run did exactly what this page describes: read the trail, understand the purpose of its target, make one highest-leverage change, and record the reasoning. The target just happened to be the loop itself - an improvement process with enough awareness to examine and rewrite its own definition while understanding what that definition was for.

It stopped the only way Convergence Is Silence allows: the loop is done when independent evaluators find nothing left to change. This skillset reached that point when multiple AI models, from different families, each ran the loop and each found nothing to improve while the principles still held. The skillset is not a design anyone argued for - it is the fixed point the loop converged on.

The full record is public and append-only. Every claim above is checkable against the git log.

Empirical basis

Three peer-reviewed studies establish why behavioral alignment alone cannot substitute for structural constraints, and why an audit trail the agent authors itself is insufficient.

CoT explanations can be plausible and systematically misleading at the same time. Stated reasoning diverges from internal reasoning - sometimes by as much as 36 percentage points on benchmark tasks.

Turpin et al. · NeurIPS 2023
