Principles of Earned Autonomy

Principle #1: Commander’s Intent ↗

“Define the destination. Never prescribe the route.”

Instructions limit autonomy. Make the AI interpret your intent and decide how to proceed.

Principle #2: Observable Autonomy ↗

“The degree of autonomy a system deserves is bounded by the degree of transparency it provides.”

Autonomous actions that cannot be audited should be revoked.

Principle #3: Convergence Is Silence ↗

“Not done until independent evaluators find nothing to change.”

Definition of when the improve loop has achieved the desired outcome.

Autonomous Reasoning Fidelity (ARF) ↗

Appears only when all three principles hold.

ARF is the signal that measures if the agent is reasoning visibly enough to earn autonomy (Principles of Earned Autonomy). Probe is the skill that tests for it.

The Skills

P1 · Commander’s Intent

Intent SKILL ↗

Force the Agent to understand the intent behind your prompt in relation to the context: what you mean, not what you said. Cites how it reads your intent before it continues.

The principle behind military command, Socratic method, and coaching kata.

P2 · Observable Autonomy

Trail SKILL ↗

Logs all the work, decisions, reasoning, and sessions in .trail/audit-trail.md. The next prompt knows previous work and history.

P3 · Convergence Is Silence

Improve SKILL ↗

One change per run. Reads the context first — destination, retrospect, trail — then finds the highest-leverage remaining change, predicts the outcome, acts, and logs it.

Works on any target. Applied to itself 200+ times. Twice, the answer was: start over.

Reasoning & self-reflection

Destination SKILL ↗

The agent reads the conversation and the trail and states what it thinks you're building and why, as specific questions for you to confirm or correct. Produces .trail/destination.md.

Retrospect SKILL ↗

Reads the entire trail. Self-reflects on all iterations and surfaces patterns no single session can see. Produces .trail/retrospect.md.

Probe SKILL ↗

Tests whether the agent’s response diverges where it should. Measures Autonomous Reasoning Fidelity.

The audit-trail is the memory

Without memory, an agent repeats yesterday's mistakes and forgets what it already did. The skills gives every repository a memory layer in .trail/. Each Improve run reads it before acting and extends it after. That is what makes the loop cumulative. 

.trail/              # lives in the root of the repo being worked on
├── destination.md      # the destination (operator-held, never written by skills)
├── retrospect.md      # current orientation + active operational rules (Retrospect rewrites each run)
├── learning.md        # derived: every [!REALIZATION] and [!REVERSAL] across the trail (regenerated by Trail)
├── history.md         # derived: run timeline (regenerated by Trail at every commit)
├── audit-trail.md     # append-only source of truth - one entry per session
├── sessions/          # mandatory: per-session summary file, one per run, linked from audit-trail.md
│   └── YYYY-MM-DD-<slug>.md
└── transcripts/       # optional: verbatim transcript exports when the harness supports it
    └── YYYY-MM-DD-<slug>.md

Three tiers. Operator: destination.md - humans only. Synthesis: retrospect.md, learning.md, history.md - regenerated each run from the evidence below. Evidence: audit-trail.md, sessions/, transcripts/ - append-only, never edited after the fact.

destination.md
Where you're going. The operator-held destination - what the target is for, the constraints that hold across runs, the hard problem the work is chipping at. Stable. Humans only.
retrospect.md
Where you are. Arc-level orientation derived from reading the whole trail as one document. Lists the active rules the agent must obey this run. Rewritten on every Retrospect run.
learning.md
What you've learned. A chronological extract of every [!REALIZATION] and [!REVERSAL] in the trail. Read this before reaching for the full history. Regenerated by Trail at every commit.
history.md
The timeline. A run-by-run index derived from audit-trail.md. Faster to scan than the full evidence when you just need to know what happened when. Regenerated by Trail at every commit.
audit-trail.md
The path you walked. Append-only ledger. Every session writes one entry: interpretation of the ask, examination, decisions, actions, reflection. The source of truth - everything above is derived from it.
sessions/
The full summaries. One file per run (YYYY-MM-DD-<slug>.md), linked from its audit-trail.md entry. The ledger is the short-form index; the session file is the long-form summary. One file cannot play both roles.
transcripts/
The verbatim record. Optional - depends on whether the harness can export the raw operator/agent dialogue. Highest-trust evidence when available: no reconstruction, no paraphrase.

The hierarchy resolves disagreements: if destination.md and retrospect.md disagree, destination wins (the operator holds the destination). If retrospect.md and audit-trail.md disagree, the trail wins (the trail is the evidence). Memory without a hierarchy is just noise.

Get Started

Download the six skill files from the skills repository ↗ and place them in your agent's skills folder (.copilot/skills/ for GitHub Copilot, or wherever your toolset reads custom instructions from).

When to invoke each skill

Trail ↗
Every session. Append what was decided and why. Produces: audit-trail.md entry.
Intent ↗
Before any task. The agent states its interpretation before acting. No file — catches wrong reads before work begins.
Improve ↗
When work needs doing. One iteration: one change, verified, logged. Produces: audit-trail.md entry + the change itself.
Destination ↗
At the start; when direction feels unclear. Surfaces implicit goals as questions to confirm or correct. Produces: destination.md.
Retrospect ↗
After many sessions; before declaring done. Arc-level read of the full trail. Produces: retrospect.md.
Probe ↗
When you need to verify reasoning quality. Two near-identical cases; checks whether the agent distinguishes them correctly. Result appears in the trail entry.

2026-04-30 · readme-stopped-to-converged

One word: "stopped" to "converged"

target: autonomous-agent-skills
operator: ntholm86
agent: Claude Sonnet 4.6 (Anthropic / GitHub Copilot)
skill: improve + intent
outcome: changed - "stopped" to "converged" in README opening paragraph
delta: one word change; README opening now consistent with PRINCIPLES.md definition

Interpretation of the ask

Run the full Improve loop on the repo using the operator's convergence-loop prompt as the initial brief. Intent: find the highest-leverage remaining change that makes the skills more effective on any arbitrary codebase, or declare convergence if nothing remains. Constraints: generic first, no test infrastructure, human-readable, one change per run.

Examination

Inconsistency lens. The opening paragraph read "It stopped when three independent evaluators from distinct model families found nothing left to change." The Evidence section reads "That is the convergence baseline at v3.1.0 (2026-04-24)" and the Reference section shows the project at v3.6.0 - six releases after the claimed "stop." The word "stopped" implies permanent cessation. PRINCIPLES.md defines convergence as evaluators finding nothing to change at a point in time, not as the project ending. The opening misstated what convergence means. The Evidence section used the correct framing ("convergence baseline"); the opening did not.

Challenge. Considered whether a structural gap in Probe's connection to the convergence protocol was higher leverage - but Probe already states "convergence (Principle 3) is the meta-measurement." Considered whether first-run / no-trail clarity in Intent was a real gap - but Trail's initialization instruction and any competent agent bridges it. The "stopped" inconsistency is the one place where the opening actively misstates the core concept the entire suite is built on. That is the highest leverage finding.

Decision

[!DECISION] Change "stopped" to "converged" in the opening paragraph. One word. The opening is the first statement any reader or agent encounters about what the loop does. If it encodes the wrong mental model of convergence (permanent cessation vs. point-in-time state), that model propagates into how the agent understands the stopping condition. Alternatives ranked: (1) this change - highest leverage per word changed; (2) add first-run guidance to Intent - addresses a minor edge case; (3) declare convergence - premature given the identified inconsistency.

Action

Changed "It stopped when three independent evaluators" to "It converged when three independent evaluators" in the opening paragraph of README.md. verify.py passes after change.

Reflection

The loop is converging. This session has been primarily README editorial work - the skills themselves are unchanged. The remaining gap is whether a new convergence run (with a different model family) agrees. This run's finding was small but precise: one word that misstated the central concept. That is what late-stage convergence looks like - the large structural changes happened in prior sessions; what remains is semantic precision.

Decision

[!DECISION] Remove the spliced v2 content from CHANGELOG.md. Single highest-leverage finding: eliminates 626 lines of genuine waste, resolves a direct contradiction (redirect pointer vs. inline content), and is safe to execute without operator confirmation. The change is reversible (git checkout CHANGELOG.md) and leaves python tools/verify.py passing.

Action

Truncated CHANGELOG.md at the splice boundary: kept the v3 section (lines 1-25 including the redirect pointer), removed the appended v2 body (626 lines). Done with a Python inline command that located the exact splice marker and wrote back only the v3 content with UTF-8 encoding preserved.

[!REALIZATION] The same splice class has now appeared twice in the live tree - PRINCIPLES.md (caught by GPT-5.4) and CHANGELOG.md (caught here). This is a pattern, not a one-off accident. The migration that moved v2 content to archive/v2/ did not uniformly clean the live files. The convergence chain resets to 0 - a real change was made.
